# 문서 5. 1차 목표(First Milestone) & 당장 첫 스텝

> **문서 유형**: 실행 기준 문서 — "생각하는 팀" → "움직이는 팀"
> **참여자**: CEO (본인) · CTO (Claude Code) · Chief of Sensemaking (ChatGPT)
> **작성일**: 2026-02-05
> **상태**: 실행 — 오늘부터 2주

---

## 0. 이 문서의 지위

- 전략 문서 ❌
- 아이디어 문서 ❌
- **실행 기준 문서 ⭕**

> **이 문서의 목적은 단 하나다: "다음 2주 동안 무엇을 하면 성공이고, 무엇을 하면 실패인가?"**

---

## 1. 1차 마일스톤 정의 (가장 중요)

### 🎯 First Milestone (Phase 0 종료 조건)

> **"우리가 풀어야 할 '단 하나의 문제 문장'을 실제 사용자 언어로 확정한다."**

이게 첫 번째 완성이다. 제품도, 매출도 아니다.

---

## 2. 성공 조건 (명확한 합격 기준)

아래 **4가지가 모두 충족**되면 성공:

| # | 조건 | 상세 |
|---|------|------|
| 1 | **인터뷰 10명 완료** | 프리랜서 ≥5, CEO/창업자 ≥5 |
| 2 | **문제 문장 후보 3개 도출** | 고객이 실제로 쓴 문장 기반, 추상어("어렵다", "복잡하다") 제거 |
| 3 | **그중 1개를 '1차 문제 정의'로 선택** | 감정 강도 높음 · 반복 빈도 높음 · 현재 대체 행동 존재 |
| 4 | **CEO가 이 문장에 서명** | "이 문제면, 최소 6개월은 판다" |

---

## 3. 실패 조건 (명확히 적는다)

아래 중 **하나라도 발생하면** 1차 마일스톤 실패:

- 인터뷰를 하다 말고 기능 논의로 넘어감
- "다 비슷한 문제인 것 같아요"로 흐림
- 프리랜서/CEO 구분 없이 하나로 뭉침
- 문제 문장을 한 문장으로 못 줄임

> 실패는 잘못이 아니다. **회피가 실패다.**

---

## 4. 2주 실행 로드맵

### Week 1 — 문제 채집 주간

| 역할 | 할 일 |
|------|-------|
| **CEO** | 인터뷰 일정 10개 확정 · 하루 최소 1명 인터뷰 · 인터뷰 직후 요약 템플릿 작성 |
| **ChatGPT** | 질문 리마인드 · 인터뷰 요약 구조화 · 문제 문장 후보 정리 보조 |
| **CTO** | 인터뷰 중 언급된 행동/도구 로그 정리 · "이 문제를 코드로 풀면 어디부터?" 메모 |

### Week 2 — 문제 압축 주간

| 역할 | 할 일 |
|------|-------|
| **CEO** | 인터뷰 요약 전체 리뷰 · 문제 문장 3개 후보 작성 · 1개 선택 및 이유 명시 |
| **ChatGPT** | 문제 문장 정제 · "이 문제를 풀면 어떤 제품이 되는가?" 논리 검증 · 문서 1·2 업데이트 지원 |
| **CTO** | 선택된 문제 기준으로 최소 기능 스케치 · 기술 난이도 평가 |

## 5. 당장 '오늘'의 첫 스텝 (가장 중요)

### 오늘 해야 할 일 (30–60분)

**① 인터뷰 대상 10명 리스트업**
- 이름
- 직업
- 연락 수단
- 가능한 날짜

**② 인터뷰 요청 메시지 보내기**
- 제품 언급 ❌
- AI 언급 ❌
- "요즘 하루 어떻게 보내는지 듣고 싶다"

**③ 캘린더에 인터뷰 슬롯 고정**
- 미루지 않게 시간 선점

> **오늘 이 3개를 하면 이 프로젝트는 이미 성공 확률이 달라진다.**
