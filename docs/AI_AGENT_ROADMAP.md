# AI Agent Vision Roadmap

> Manage Agent App: AI 비서 → 중앙 관리 에이전트로 확장하기 위한 로드맵

**작성일**: 2026-02-02

---

## 1. 비전 요약

이 앱은 개인 생산성 관리(Plan → Focus → Review)를 기반으로 시작해, 다양한 앱과 물리적 AI/디바이스까지 통합 관리하는 **중앙 AI 에이전트**로 확장한다.

핵심 목표는 **"컨텍스트를 이해하고, 실제 행동을 대신 수행하며, 안전하게 승인 흐름을 갖춘 AI 비서"**다.

---

## 2. 장기 로드맵 (큰 그림)

### Phase 0 — 기반 정리
- 데이터 품질/스키마 안정화
- AI 호출 모니터링 (비용, 오류, 성능)
- 인간 승인(Approval) 구조 설계
- 감사 로그/변경 이력 기반 확보

### Phase 1 — 개인 비서
- 자연어 입력/요약/추천
- 목표/작업 자동 분해
- 주간 계획 자동 생성
- 회고 자동 요약

### Phase 2 — 행동 가능 에이전트
- 캘린더/이메일/노션 등 외부 앱 연동
- 작업 생성/수정/이동을 AI가 직접 수행
- 승인 흐름으로 안전하게 실행

### Phase 3 — 멀티 에이전트
- 목표/업무 유형별 서브 에이전트 분리
- 충돌/우선순위 조정 메커니즘
- 장기 목표 기반 조정

### Phase 4 — 중앙관리자 (Physical + Digital)
- 스마트홈/웨어러블/디바이스 통합
- 환경 + 일정 + 컨디션을 통합한 실행
- 사용자 행동/상황 변화에 자동 적응

## 3. 지금 앱에서 “관리(Management)” 기능을 완성하는 흐름

현재의 Plan 영역을 "관리 도구"에서 "AI 의사결정 지원"으로 확장한다.

### Milestone A — 관리 기본기 완성
- 목표/작업/루틴/캘린더 **일관된 연결**
- 의존성/우선순위/기간 관리
- 반복 작업 자동 생성 + 성과 추적
- 일정 충돌 감지 및 기본 재배치 기능

### Milestone B — AI가 계획을 생성
- 목표 입력 → 작업/마일스톤 자동 분해
- 주간 계획 자동 생성 (빈 슬롯 기준 배치)
- 충돌 발생 시 대안 스케줄 제안

### Milestone C — 실행 최적화
- 집중 세션 추천 (패턴 기반)
- 작업 시작 전에 “다음 1스텝” 제안
- 지연 사유/중단 흐름 자동 기록

### Milestone D — 회고 & 개선 자동화
- 주간 회고 자동 요약
- 지연/성과 패턴 분석
- 다음 주 목표 현실화 제안

## 4. 지금 당장 Management에서 확장할 기능 우선순위

1) 작업 의존성 + 블로킹 관계
2) 일정 충돌 감지 + 대안 배치
3) 목표 → 작업 자동 분해 (AI)
4) 주간 계획 자동 생성 (AI)
5) 목표/작업 건강지수 (지연 위험도)
6) 계획 변경 히스토리 + 롤백
7) 승인 흐름 (AI가 만든 계획 → 사용자 승인)

---

## 5. 중앙관리자 확장을 위한 설계 원칙

### Action Layer
- AI가 외부 앱에 실제 변경을 수행할 수 있어야 함
- 모든 행동은 승인/로그 기반

### Context Layer
- 일정, 환경, 사용자 상태를 통합
- 정적 입력뿐 아니라 패턴 기반 추론 필요

### Approval Layer
- 실행 전 검토 가능한 UI 제공
- 위험도/변경 범위에 따라 단계적 승인

### Memory Layer
- 사용자 패턴/선호를 장기적으로 학습
- 반복 업무는 자동화 수준을 점진적으로 확대
